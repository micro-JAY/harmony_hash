## ADDED Requirements

### Requirement: Branded browser assets
The application shell SHALL expose the current Harmony Hash identity through stable favicon and Apple touch-icon links that remain usable by supported browsers and pinned mobile shortcuts.

#### Scenario: Document branding links
- **WHEN** the application document is parsed
- **THEN** it SHALL include a PNG favicon link and an Apple touch-icon link that resolve to the current canonical Harmony Hash asset derivatives
